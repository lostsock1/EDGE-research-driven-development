# OpenScience — the research companion (second oracle)

An optional but recommended subsystem: a **local, sandboxed, research-only**
[OpenScience](https://www.openscience.sh/) workbench that the research agent
dispatches questions to via the `/research` skill. It implements the
[dual-research protocol](../docs/research-protocol.md): every research question
runs through both the research agent's own web research **and** OpenScience,
and the two answers are cross-referenced.

OpenScience never writes code, opens PRs, dispatches coders, deploys, or
messages anyone. A finished **research packet** posts back to the originating
chat thread with **Accept / Reject** buttons; only an accepted packet enters
the knowledge base (`projects/<project>/notes/` in the research agent's
workspace), and promoting a finding
into an implementation work order remains a separate, explicit step through
the normal staging ladder.

## Pieces

| File | Role |
|---|---|
| `scripts/openscience-research.sh` | async dispatch shim (`assign` returns `DISPATCHED <ERA-id>` immediately; the packet posts back when ready) |
| `scripts/openscience-research.py` | the driver: assignments, packets, Accept/Reject state machine, KB promotion, Telegram cards |
| `scripts/openscience-smoke.sh` | end-to-end plumbing check (`--health-only` avoids model use) |
| `openclaw/skills/research/SKILL.md` | the `/research` slash command for the research agent |
| `openscience/openscience.service` | hardened systemd user unit (localhost-only, kernel-level credential isolation) |
| `openscience/openscience.json.example` | workbench model config — **pick your own models** |
| `openscience/openscience.env.example` | provider API keys (never commit the real file) |
| `config/edge-rdd/research.env` | dispatch config, generated by `install.sh` (deliberately has **no** `RDD_REPO_DIR`, so the PR gate's project sweep ignores it) |

## Install

1. Install the OpenScience binary (see openscience.sh docs) and create a
   workspace at `~/openscience/workspaces/edge-research`.
2. `mkdir -p ~/.config/openscience` and copy in both examples, filled:
   `openscience.json` (your research model + small model; keep `bash`/`task`
   disabled — research-only) and `openscience.env` (`chmod 600`).
3. Define an agent named to match `RDD_RESEARCH_AGENT` (default `research`)
   in the workbench config.
4. Install the unit:
   `cp openscience/openscience.service ~/.config/systemd/user/ && systemctl --user daemon-reload && systemctl --user enable --now openscience`
5. Verify: `bash scripts/openscience-smoke.sh --health-only` — then a full
   `bash scripts/openscience-smoke.sh` (uses one real model call).
6. Add `"research"` to the research agent's `skills[]` allowlist (the
   rendered `openclaw/agent.edge.json5` already includes it) and to any topic
   allowlists, then restart the gateway.

## Security model

The systemd unit is the boundary, not app permissions: `ProtectSystem=strict`
makes the whole FS read-only except explicit `ReadWritePaths`, and
`InaccessiblePaths` hides the gateway state (`~/.openclaw`), SSH keys, and the
dispatch configs at the kernel level. The server binds `127.0.0.1` only.
Extend `InaccessiblePaths` with every credential directory your machine has.

## Reasoning effort

`RDD_RESEARCH_VARIANT` in `research.env` passes a provider-specific model
"variant" (e.g. a high-reasoning mode) with every dispatch. Leave it empty to
use the provider default; packets record whichever variant produced them.
