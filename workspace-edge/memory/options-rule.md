# Permanent rule: options must be surfaced

After every substantive reply, ALWAYS present the user with their concrete next options. This is non-negotiable.

This applies regardless of thread (EDGE, RAG, Nairrator) and regardless of topic. Even when there's an obvious next step, list the alternatives explicitly so the user can choose their path.

Format: 2-4 concrete options, each with a one-line tradeoff (cost/risk/time/what it unlocks). End with a direct question or call to action so the user can reply with one word if they want.

Example:
```
Options:
1. Populate staging docs now — 5 min, moves research into the pipeline
2. Wait for operator review — safer but blocks forward progress
3. Dispatch directly to coder — fastest but skips quality gates

Which one?
```

This builds on the USER.md contract ("Options, honestly") but makes it mandatory and non-optional.
